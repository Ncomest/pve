# Уровень вещи (Item Level) и система лута

## Назначение

- **Item level** — уровень экземпляра вещи. Чем выше уровень босса, тем выше item level падающей вещи, тем больше эффективные статы (при той же комбинации характеристик).
- **Экземпляр вещи (ItemInstance)** — конкретная «копия» шаблона с фиксированным уровнем. В инвентаре и экипировке хранятся экземпляры, а не ссылки на шаблоны. Это позволяет одной и той же модели вещи (например, «Кольцо силы») иметь разный уровень и разные статы в зависимости от того, с какого босса она выпала.
- **Шаблон (ItemTemplate)** — описание вещи: слот, название, редкость, базовые статы (ровно 3 характеристики). База содержит 10 слотов × 10 шаблонов = 100 шаблонов с разными комбинациями статов.

## Формулы

- **Уровень вещи при дропе:**  
  `itemLevel = 1 + bossLevel * 3`  
  Пример: босс 1 ур. → item level 4, босс 2 ур. → 7.

- **Множитель статов:**  
  `multiplier = 1 + (itemLevel - 1) * 0.1`  
  Пример: item level 5 → множитель 1.4; база HP 5 → эффективно 7.

- **Эффективные статы:**  
  Для числовых статов (hp, power, speed, armor): `effective = round(base * multiplier)`.  
  Для долей (chanceCrit, evasion): умножить на multiplier и при необходимости ограничить сверху (например, максимум 0.5).

## Структуры данных

- **ItemTemplate:** `id`, `name`, `slot`, `rarity`, `baseStats` (объект ровно с 3 полями из hp/power/chanceCrit/evasion/speed/armor).
- **ItemInstance:** `instanceId` (уникальный id экземпляра), `templateId`, `itemLevel`. Сериализуется в localStorage как есть; при загрузке по `templateId` поднимается шаблон и пересчитываются эффективные статы.
- **Item (отображаемый вид):** `id` (= instanceId для уникальности в UI), `name`, `slot`, `rarity`, `stats` (уже эффективные), `itemLevel`. Получается из экземпляра через `getDisplayItem(instance, templatesMap)`.

## Правила дропа

- С босса за один бой падает **ровно 2 вещи**.
- Обе выбираются **случайно** из общего пула всех шаблонов (100 шт.), без повтора в рамках одного дропа.
- Уровень каждой вещи: `itemLevel = 1 + bossLevel * 3`.
- При победе создаются два `ItemInstance` с новыми `instanceId` (uuid) и добавляются в список лута; игрок забирает их в инвентарь.

## Миграция старых сохранений

При первой загрузке состояния из localStorage проверяем каждый элемент в `inventory` и `equipped`:

- Если у элемента есть поле `stats` и **нет** поля `templateId` — считаем его старым форматом.
- Миграция: `templateId = item.id`, `itemLevel = 1`, генерируем новый `instanceId`. Заменяем объект на `ItemInstance`.

После миграции все операции идут только с экземплярами.
