# Уровень вещи (Item Level) и система лута

## Назначение

- **Item level** — уровень экземпляра вещи. Чем выше уровень босса, тем выше item level падающей вещи, тем больше эффективные статы (при той же комбинации характеристик).
- **Экземпляр вещи (ItemInstance)** — конкретная «копия» с фиксированным уровнем. Может содержать `generatedBaseStats` и `rarityOverride` при процедурной генерации с дропа/магазина.
- **Шаблон (ItemTemplate)** — описание: слот, название, редкость по умолчанию, опционально устаревшие `baseStats` для старых сохранений.

## Формулы

- **Уровень вещи при дропе с обычного босса:**  
  `itemLevel = 1 + bossLevel * 3`
- **Множитель статов:**  
  `multiplier = 1 + (itemLevel - 1) * 0.1`
- **Эффективные статы:** как в `getEffectiveStats` — база × множитель уровня × роллы; при наличии `generatedBaseStats` на экземпляре используется она вместо `template.baseStats`.

## Редкость и веса

Редкости: **common** (белая), **uncommon** (зелёная), **rare** (синяя), **epic** (фиолетовая), **unique** (жёлтая).

Шанс при дропе и при генерации витрины торговца:

| Редкость   | Шанс |
|-----------|------|
| common    | 90%  |
| uncommon  | 4%   |
| rare      | 3%   |
| epic      | 2%   |
| unique    | 1%   |

### Ролл силы линий статов (процедурный лут)

При генерации `generatedBaseStats` в `generateBaseStatsForRarity` **каждая** числовая линия (HP, броня, дух и т.д.) получает **свою** случайную долю силы в пределах диапазона редкости — не одну общую на всю вещь.

| Редкость   | Доля силы ролла (min…max от «базы» `PROC_BASE`) |
|-----------|--------------------------------------------------|
| common    | 10–20%                                           |
| uncommon  | 21–40%                                           |
| rare      | 41–60%                                           |
| epic      | 61–80%                                           |
| unique    | 81–100%                                          |

Базовые «единицы» масштаба по типам статов заданы в `PROC_BASE` в `src/entities/item/lib/itemGeneration.ts`. Значение линии: `max(1, round(PROC_BASE[key] * strength))`, где `strength` — независимый ролл для этой линии.

**Зачем отдельный ролл на линию:** если бы один множитель применялся ко всем статам сразу, у характеристик с **одинаковой** базой в `PROC_BASE` (например броня и дух — обе 50) после округления всегда получались бы **одинаковые** числа на экране. Независимые роллы убирают этот артефакт.

Наборы того, **какие** линии вообще появляются на вещи, по-прежнему задаются процедурно по редкости в той же функции `generateBaseStatsForRarity`.

## Правила дропа

- **Обычный босс:** за победу падает **ровно 2** вещи (случайные шаблоны слотов из пула экипировки, для каждой вещи генерируются редкость и статы).
- **Босс-ресурс** (записи из `resources.json`): **только ресурсы**, **ровно 1** случайный ресурс за бой (без экипировки).
- Уровень вещи по-прежнему `1 + bossLevel * 3` для экипировки.

## Миграция старых сохранений

Старые экземпляры без `generatedBaseStats` используют `template.baseStats` из шаблона.
