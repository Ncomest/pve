# Баланс статов вещей

Где менять числовые значения, если нужно подкрутить силу лута, отображение статов и их влияние в бою.

---

## 1. Базовые статы вещей (сколько даёт одна характеристика на предмете)

**Файл:** `src/entities/item/items-db.ts`

- **`BASE`** — базовые значения для каждого типа стата. У каждой вещи 3 стата из набора (см. `STAT_COMBOS`).
  - `hp`, `power`, `speed`, `armor` — целые числа, отображаются как есть (например, +5 HP, +1 атака, +10 брони).  
    В боевой формуле они дополнительно преобразуются (проценты скорости/брони и доп. вклад power в урон).
  - `chanceCrit`, `evasion` — **очки**; на вещи показываются как «50 крита», «25 уклонения»; в статах персонажа переводятся в проценты через коэффициенты из п. 2.
- **`STAT_COMBOS`** — какие комбинации из 3 статов выдаются шаблонам (менять при добавлении новых статов в `BASE`).

---

## 2. Вес статов «в процентах» (крит, уклонение, скорость, броня): очки → %

**Файл:** `src/entities/item/lib/statPoints.ts`

- **`CRIT_POINTS_TO_FRACTION`** — сколько даёт одно очко крита в доле (0..1). Формула: `(нужный_процент / 100) / кол-во_очков`.  
  Пример: 50 очков = 0.25% → `0.0025 / 50`.
- **`EVASION_POINTS_TO_FRACTION`** — то же для уклонения.  
  Пример: 25 очков = 0.25% → `0.0025 / 25 = 0.0001`.
- **`SPEED_POINTS_TO_FRACTION`** — перевод очков скорости в долю для отображения в UI (страница Персонажа).  
  Пример: 50 очков скорости = 0.25%.
- **`ARMOR_POINTS_TO_FRACTION`** — перевод очков брони в долю снижения урона.  
  Пример: 50 очков брони = 1% снижения входящего урона, 500 брони ≈ 10%. Значение ограничено сверху (в коде сейчас максимум 90%).

Использование:

- при суммировании статов с экипировки в `src/app/store/character.ts` очки крита/уклонения умножаются на эти коэффициенты и ограничиваются 1 (100%);
- скорость и броня в UI страницы Персонажа выводятся в процентах через эти функции;
- в боевой формуле броня используется как процент снижения урона: `damage = raw * (1 - armorPercent)`.

---

## 3. Влияние уровня вещи на статы

**Файл:** `src/entities/item/lib/itemLevel.ts`

- **`getStatMultiplier(itemLevel)`** — множитель от уровня: сейчас `1 + (itemLevel - 1) * 0.1` (уровень 5 → множитель 1.4).  
  Менять коэффициент `0.1`, если нужно усилить или ослабить рост статов от уровня вещи.

---

## 4. Влияние атаки (power) на урон

**Файл с формулой урона:** `src/features/battle/model/useBattle.ts` (`calcHit`).

- Базовый урон рассчитывается как:
  - берётся значение `power` атакующего;
  - умножается на случайный множитель от `0.9` до `1.1`;
  - при крите применяется множитель крита (по умолчанию 1.5).
- Дополнительный плоский бонус от силы:
  - каждые **25 единиц `power` дают +1 к урону** поверх базового значения;
  - формула: `flatFromPower = floor(power / 25)`, далее `raw = baseWithCrit + flatFromPower`.
- Далее к `raw` применяется процентное снижение урона от брони (`ARMOR_POINTS_TO_FRACTION`).

Таким образом, рост `power` одновременно:

- линейно увеличивает базовый урон;
- даёт дополнительный плоский бонус каждые 25 очков.
