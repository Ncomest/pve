# Баланс статов вещей

Где менять числовые значения, если нужно подкрутить силу лута или отображение статов.

---

## 1. Базовые статы вещей (сколько даёт одна характеристика на предмете)

**Файл:** `src/entities/item/items-db.ts`

- **`BASE`** — базовые значения для каждого типа стата. У каждой вещи 3 стата из набора (см. `STAT_COMBOS`).
  - `hp`, `power`, `speed`, `armor` — целые числа, отображаются как есть (например, +5 HP, +1 атака).
  - `chanceCrit`, `evasion` — **очки**; на вещи показываются как «50 крита», «25 уклонения»; в статах персонажа переводятся в проценты через коэффициенты из п. 2.
- **`STAT_COMBOS`** — какие комбинации из 3 статов выдаются шаблонам (менять при добавлении новых статов в `BASE`).

---

## 2. Вес статов «в процентах» (крит, уклонение): очки → %

**Файл:** `src/entities/item/lib/statPoints.ts`

- **`CRIT_POINTS_TO_FRACTION`** — сколько даёт одно очко крита в доле (0..1). Формула: `(нужный_процент / 100) / кол-во_очков`.  
  Пример: 50 очков = 0.25% → `0.0025 / 50`.
- **`EVASION_POINTS_TO_FRACTION`** — то же для уклонения.  
  Пример: 25 очков = 0.25% → `0.0025 / 25 = 0.0001`.

Использование: при суммировании статов с экипировки в `src/app/store/character.ts` очки крита/уклонения умножаются на эти коэффициенты и ограничиваются 1 (100%). Добавление новых «очковых» статов потребует правок здесь и в store.

---

## 3. Влияние уровня вещи на статы

**Файл:** `src/entities/item/lib/itemLevel.ts`

- **`getStatMultiplier(itemLevel)`** — множитель от уровня: сейчас `1 + (itemLevel - 1) * 0.1` (уровень 5 → множитель 1.4).  
  Менять коэффициент `0.1`, если нужно усилить или ослабить рост статов от уровня вещи.
